# Ignite Brief Template

A seed brief written from ignition intent capture. It exists so `bmad-prd` starts from what the user already said instead of a blank page. Keep it honest and small: fill only what the intent conversation actually established, and put everything else under Open Questions. Drop sections that have nothing real in them.

## Default Structure

```markdown
---
generated_by: bmad-ignite
status: seed
date: {date}
---

# Project Brief: {Project Name}

> Seeded by bmad-ignite from intent capture. A starting point for `bmad-prd`
> or a full `bmad-product-brief` pass — not a finished brief.

## Summary

[2-3 sentences: what this is, the problem it addresses, why now. Only what the user actually said.]

## Who This Serves

[Audience and domain from intent capture. Primary users and what success looks like for them.]

## Capability Needs

[The needs the user confirmed — auth, payments, realtime, AI features — and what they declined or deferred.]

## Scope Signals

[Anything the user said about what is in or out for the first version. If nothing was said, delete this section.]

## Open Questions

[Everything a PRD will need that intent capture did not establish. Be specific — these become the first discovery questions downstream.]
```
